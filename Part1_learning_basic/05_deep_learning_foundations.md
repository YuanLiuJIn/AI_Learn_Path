# 05 深度学习基础

深度学习是大模型的直接基础。大语言模型、扩散模型、视觉模型、多模态模型，本质上都是大规模神经网络。

## 1. 神经网络的基本直觉

传统模型往往需要人手工设计特征。

深度学习希望模型自己从数据中学习表示。

例如图片识别：

- 低层学习边缘、颜色。
- 中层学习纹理、局部形状。
- 高层学习物体部件和整体语义。

对于文本：

- 低层可能学习词形、局部搭配。
- 高层学习句法、语义、任务相关模式。

### 网络结构：输入层、隐藏层、输出层

一个神经网络从左到右分成三类层：

```text
输入层 -> 隐藏层（可以有很多层）-> 输出层
```

以“判断图片是猫还是狗”为例：

- 输入层：原始数据入口，不做计算。例如图片的像素、表格样本的特征。
- 隐藏层：夹在中间、负责加工信息的层。“隐藏”指它既不是输入也不是最终输出，平时不直接看它的值。层数越多越“深”，这就是深度学习的“深度”。每个隐藏层做的就是 `z = W*x + b` 再接激活函数。
- 输出层：最后一层，给出答案，形式取决于任务：

```text
二分类  -> 1 个输出 + Sigmoid -> 输出"是猫的概率"
多分类  -> 多个输出 + Softmax  -> 多个类别概率，和为 1
回归    -> 1 个输出，不加激活  -> 直接输出数值
```

每条连接都有一个权重 `w`，这些权重就是模型要学习的东西。

## 2. 神经元

一个最简单的神经元可以写成：

```text
z = w1*x1 + w2*x2 + ... + wn*xn + b
a = activation(z)
```

它先做线性变换，再通过激活函数引入非线性。

如果没有激活函数，多层线性网络仍然等价于一层线性模型。

## 3. 张量

张量是深度学习中的基本数据结构。

| 维度 | 名称 | 例子 |
|---|---|---|
| 0 维 | 标量 | 一个损失值 |
| 1 维 | 向量 | 一个词向量 |
| 2 维 | 矩阵 | 一批样本的特征 |
| 3 维及以上 | 张量 | 图片 batch、文本 batch |

常见形状：

```text
表格数据: [batch_size, features]
图片数据: [batch_size, channels, height, width]
文本数据: [batch_size, sequence_length]
词嵌入: [batch_size, sequence_length, hidden_size]
```

大模型中的很多错误都来自 shape 理解不清。

## 4. 前向传播

前向传播是“用当前的参数做一次预测”的过程：数据从输入层一层层往右走，最后得到输出。

```text
输入 x -> 第 1 层 -> 第 2 层 -> ... -> 输出 y_hat
```

例如一个 MLP：

```text
h1 = ReLU(W1*x + b1)
h2 = ReLU(W2*h1 + b2)
y_hat = W3*h2 + b3
```

用具体数字走一遍（简化为单特征，输入 `x = 2`）：

```text
输入层：x = 2

隐藏层（设 w=3, b=1）：
  z = 3*2 + 1 = 7
  a = ReLU(7) = 7

输出层（设 w=2, b=0）：
  y_hat = 2*7 + 0 = 14
```

方向是：输入 2 -> 隐藏层 7 -> 输出 14。这就是前向传播：拿现在的参数，算出一个预测值。

## 5. 损失函数

损失函数衡量模型输出和真实答案的差距。

常见损失：

- 回归：MSE、MAE。
- 分类：Cross Entropy。
- 语言模型：next token prediction 的 Cross Entropy。

训练神经网络就是让损失函数尽量变小。

## 6. 反向传播

反向传播用于计算每个参数对损失的影响，方向和前向传播相反：从输出端的损失往输入端逐层回推。

```text
前向传播：输入 ──────> 输出   （算预测）
反向传播：输入 <────── 损失   （算每个参数的责任）
```

核心思想是“追责”：损失大说明有参数没调好，反向传播从输出往回，算出每个参数对这个错误负多少责任。这个“责任”就是梯度，用的数学工具是链式法则（一层套一层求导）。

直觉类比：一条流水线产出了次品（损失大），就从最后一道工序往前查，逐个工序算它贡献了多少错误，从而知道每个环节该怎么调整。

四个角色配合：

- 前向传播：算预测。
- 损失函数：算错了多少。
- 反向传播：算每个参数应该怎么改（梯度）。
- 优化器：按梯度真正更新参数（`参数 = 参数 - 学习率 × 梯度`）。

在 PyTorch 中，`loss.backward()` 这一行就自动完成了反向传播的全部计算（即 autograd 的作用）。

## 7. 梯度下降与优化器

最基本更新公式：

```text
参数 = 参数 - 学习率 × 梯度
```

常见优化器：

| 优化器 | 特点 |
|---|---|
| SGD | 简单、泛化常较好，但调参要求高 |
| Momentum | 加入动量，减少震荡 |
| RMSProp | 自适应学习率 |
| Adam | 常用，收敛快 |
| AdamW | 深度学习和大模型训练常用，解耦 weight decay |

大模型训练中 AdamW 非常常见。

## 8. 激活函数

### 8.1 为什么必须有激活函数

神经元做两件事：

```text
第一步（线性）：z = w1*x1 + ... + wn*xn + b
第二步（非线性）：a = activation(z)
```

如果没有第二步，把多层线性层叠起来：

```text
h1 = W1 * x
h2 = W2 * h1 = (W2 * W1) * x
```

`W2 * W1` 仍然是一个矩阵，等价于一层线性模型。也就是说：

> 没有非线性激活，不管堆多少层，整个网络仍然只能表达线性关系，画不出曲线。

激活函数的意义就是：让“深度”变得有意义，让网络能逼近复杂函数。

直觉类比：

```text
线性 = 只能用直尺画图
非线性激活 = 给你一支能画曲线的笔
```

### 8.2 常见激活函数

| 激活函数 | 输出范围 | 优点 | 缺点 | 常用位置 |
|---|---|---|---|---|
| Sigmoid | 0 ~ 1 | 可解释为概率 | 两端饱和、梯度消失、非零中心 | 二分类输出层 |
| Tanh | -1 ~ 1 | 零中心，比 Sigmoid 好 | 仍会梯度消失 | RNN、早期网络 |
| ReLU | 0 ~ +∞ | 计算快、缓解梯度消失 | 神经元死亡 | 隐藏层默认选择 |
| Leaky ReLU | -∞ ~ +∞ | 缓解神经元死亡 | 多一个超参数 | 隐藏层 |
| GELU | 含少量负值 | 平滑、效果好 | 计算略复杂 | Transformer |
| SiLU / Swish | 含少量负值 | 平滑、效果好 | 计算略复杂 | 现代网络 |

### 8.3 几个关键直觉

- Sigmoid / Tanh：`z` 很大或很小时曲线很平，导数接近 0，反向传播梯度越乘越小，造成**梯度消失**，所以现在很少用在隐藏层。
- ReLU：`ReLU(z) = max(0, z)`，正区间导数恒为 1，缓解梯度消失，是隐藏层默认选择。
- Dead ReLU（神经元死亡）：如果某神经元的 `z` 一直 <= 0，输出和梯度都为 0，它就再也学不动了。Leaky ReLU 给负区间一个小斜率（如 `0.01*z`）来缓解。
- GELU / SiLU：可以理解为 ReLU 的“平滑版”，在 0 附近不是硬拐弯，而是平滑过渡，Transformer 和大模型中很常见。

### 8.4 怎么选

```text
隐藏层：默认 ReLU；出现神经元死亡可换 Leaky ReLU
Transformer / 大模型：GELU 或 SwiGLU
二分类输出层：Sigmoid
多分类输出层：Softmax（输出归一化，多个概率之和为 1）
回归输出层：通常不加激活，直接输出实数
```

注意区分**隐藏层激活**（负责提供非线性）和**输出层激活**（取决于任务类型）。

## 9. 梯度消失与梯度爆炸

### 梯度消失

梯度在反向传播中越来越小，前面层几乎学不到。

常见原因：

- 网络太深。
- 激活函数饱和。
- 初始化不合适。

### 梯度爆炸

梯度变得非常大，训练不稳定，损失可能变成 NaN。

常见解决方法：

- 合理初始化。
- 归一化层。
- 残差连接。
- 梯度裁剪。
- 调小学习率。

## 10. 正则化与泛化

深度学习中常见防过拟合方法：

- 更多数据
- 数据增强
- weight decay
- dropout
- early stopping
- batch normalization
- label smoothing

大模型场景中，数据规模、去重、质量控制和训练策略都影响泛化能力。

## 11. Batch Normalization 与 Layer Normalization

### Batch Normalization

对 batch 维度做归一化，常用于 CNN。

### Layer Normalization

对特征维度做归一化，常用于 Transformer。

大语言模型中更常见 LayerNorm 或 RMSNorm。

## 12. Dropout

Dropout 是一种防止过拟合的技巧：训练时**随机让一部分神经元临时“罢工”**（输出置为 0），每次罢工的神经元都不一样。

```text
正常：   [神经元1] [神经元2] [神经元3] [神经元4]
Dropout：[神经元1] [✗罢工 ] [神经元3] [✗罢工 ]   （随机丢弃部分）
```

为什么有效：

- 不让模型过度依赖某几个神经元。一旦某个被丢弃，模型得学会用别的神经元也能判断，逼着每个神经元都学到有用的东西。类比一个团队不能只靠一个人，强制随机有人请假，逼大家都有本事，整体更稳健。
- 相当于训练了很多个略有不同的子网络，最终效果接近它们的“平均”，更不容易过拟合。

关键：训练和测试行为不同。

```text
训练时：随机丢弃神经元（Dropout 生效）
测试时：不丢弃，用全部神经元（Dropout 关闭）
```

所以 PyTorch 中要正确切换模式：`model.train()` 训练、`model.eval()` 评估。如果评估时忘了 `model.eval()`，Dropout 还在随机丢弃，结果会不稳定——这是新手常见坑。

## 13. MLP、CNN、RNN、Transformer 的位置

本部分只打基础，后续会系统学习网络结构。这里先建立地图：

| 结构 | 适合数据 | 核心思想 |
|---|---|---|
| MLP | 表格、简单特征 | 全连接层堆叠 |
| CNN | 图像、局部结构 | 卷积提取局部模式 |
| RNN/LSTM/GRU | 序列数据 | 递归处理时间序列 |
| Transformer | 文本、多模态 | Attention 建模全局关系 |

大模型主要基于 Transformer 及其变体。

## 14. 深度学习训练流程

标准流程：

```text
准备数据
  -> 构建模型
  -> 定义损失函数
  -> 选择优化器
  -> 前向传播
  -> 计算损失
  -> 反向传播
  -> 更新参数
  -> 验证模型
```

PyTorch 中对应：

```text
model(x)
loss_fn(output, y)
loss.backward()
optimizer.step()
optimizer.zero_grad()
```

## 15. 与大模型的关系

| 深度学习基础 | 大模型中的对应 |
|---|---|
| 张量 | token embedding、attention matrix |
| 反向传播 | 训练数十亿参数 |
| 优化器 | AdamW、学习率调度 |
| 归一化 | LayerNorm、RMSNorm |
| 激活函数 | GELU、SwiGLU |
| 正则化 | weight decay、dropout、数据去重 |
| GPU | 分布式训练、显存优化 |

## 本章小结

深度学习的核心是用多层可微函数从数据中学习表示。你需要重点理解张量、前向传播、损失函数、反向传播和优化器。掌握这些后，学习 Transformer 和大语言模型会容易很多。
