# 00 学习路线

## 总体目标

本阶段的目标不是“记住所有算法”，而是建立一个稳定的机器学习知识框架：

```text
数据 -> 模型 -> 损失函数 -> 优化算法 -> 评估指标 -> 泛化能力 -> 工程实现
```

大模型也是这个框架的延伸，只是模型更大、数据更多、训练更复杂、工程系统更庞大。

## 建议 8 周学习计划

### 第 1 周：机器学习基础

重点：

- 什么是机器学习
- 监督学习、无监督学习、强化学习
- 训练集、验证集、测试集
- 过拟合、欠拟合、泛化能力
- 回归、分类、聚类

产出：

- 能用自己的话解释机器学习和传统编程的区别。
- 能说明为什么不能只看训练集准确率。

### 第 2 周：数学与线性模型

重点：

- 向量、矩阵、张量
- 均值、方差、概率分布
- 损失函数
- 梯度下降
- 配套阅读 `09_math_foundations_for_ml.md`
- 线性回归、逻辑回归
- L1/L2 正则化

产出：

- 能手写线性回归的预测公式。
- 能解释为什么梯度下降可以让损失变小。

### 第 3 周：决策树与 CART

重点：

- 决策树的直觉
- 信息增益、基尼指数、均方误差
- CART 分类树和回归树
- 剪枝

产出：

- 能解释一棵树如何从特征中做判断。
- 能说清楚分类树和回归树的区别。

### 第 4 周：随机森林与 GBDT

重点：

- Bagging 与 Boosting
- 随机森林
- GBDT
- XGBoost、LightGBM、CatBoost 的基本差异

产出：

- 能解释随机森林为什么降低方差。
- 能解释 GBDT 为什么是逐步修正错误。

### 第 5 周：传统机器学习实践

重点：

- 数据清洗
- 特征工程
- 交叉验证
- 指标选择
- 模型对比

产出：

- 完成一个表格数据预测项目。
- 至少比较线性模型、决策树、随机森林、GBDT。

### 第 6 周：深度学习基础

重点：

- 神经元、层、激活函数
- 前向传播、反向传播
- MLP
- 优化器
- Batch Normalization、Dropout

产出：

- 能解释神经网络为什么可以拟合复杂函数。
- 能写出一个简单 MLP 的训练流程。

### 第 7 周：GPU 与 CUDA

重点：

- CPU 与 GPU 的区别
- 并行计算
- CUDA 线程模型
- 显存、带宽、吞吐
- 为什么深度学习依赖矩阵乘法

产出：

- 能解释为什么 GPU 适合训练神经网络。
- 能看懂 PyTorch 中张量放到 `cuda` 的意义。

### 第 8 周：PyTorch

重点：

- Tensor
- autograd
- nn.Module
- Dataset/DataLoader
- training loop
- 模型保存与加载
- GPU 训练

产出：

- 用 PyTorch 完成一个分类任务。
- 能独立排查张量 shape、device、dtype 的常见错误。

## 每章推荐学习方法

### 1. 先用一句话解释

例如：

- 线性回归：用一条线或一个平面拟合数据趋势。
- 决策树：像做选择题一样一步步缩小答案范围。
- 随机森林：让很多棵不一样的树投票，减少单棵树的偶然性。
- GBDT：后一棵树专门修正前面模型犯的错。

### 2. 再看公式

公式的作用是把直觉变成可计算的规则。不要一开始就陷入符号。

### 3. 最后写代码

代码不是抄 API，而是验证你是否理解：

- 输入是什么？
- 输出是什么？
- 损失函数是什么？
- 参数如何更新？
- 指标如何判断？

## Part1 最小完成标准

完成以下内容即可进入下一阶段：

- 阅读所有章节。
- 完成 `08_practice_projects.md` 中至少 4 个项目。
- 能不用查资料解释：过拟合、梯度下降、CART、随机森林、GBDT、反向传播、CUDA、autograd。
- 能看懂 `Xw + b`、交叉熵、mini-batch、张量 shape、dtype、device 的含义。
- 能用 PyTorch 写出一个完整训练循环。
