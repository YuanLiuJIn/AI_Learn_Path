# 06. Hermes vs OpenClaw：两种哲学的选择

> 目标：对比 Hermes 和 OpenClaw 的设计哲学、适用场景和选型建议。
> 这不是"谁更好"的问题，而是"哪个更适合你的场景"的问题。

## 0. 两种哲学

```
OpenClaw 的哲学：连接一切，本地掌控
  → 给模型最多的工具
  → 让用户拥有精致的多端体验
  → 多 Agent 协作是核心能力

Hermes 的哲学：越用越懂你，自我进化
  → 给 Agent 记忆和学习能力
  → 让 Agent 在陪伴中成长
  → 长寿命单 Agent 是核心能力
```

## 1. 全面对比

### 核心差异

| 维度 | OpenClaw 🦞 | Hermes ☤ |
|---|---|---|
| **定位** | 多端精致体验的个人 AI 工作台 | 会自我进化的长寿命 AI 助手 |
| **核心理念** | 连接一切，本地掌控 | 用→学→进化 |
| **目标用户** | 想要多端精致体验的开发者/产品用户 | 想要 Agent 自主学习的工程师/研究者 |

### 记忆系统

| 维度 | OpenClaw | Hermes |
|---|---|---|
| 记忆持久化 | 插件化，手动配置 | 多后端自动持久化，主动沉淀用户画像 |
| 记忆写入 | 手动或被动 | Background Review 自动复盘写入 |
| 记忆检索 | 按需加载 | 双通路：内置快照 + 外部语义检索 |
| 用户画像 | 无内置 | USER.md 自动积累 |

### 技能管理

| 维度 | OpenClaw | Hermes |
|---|---|---|
| 技能创建 | 用户或 skill-creator 显式编写 | Agent 可**自主创建**新技能 |
| 技能分发 | ClawHub 市场 | 本地管理 |
| 生命周期 | 无自动管理 | Curator 自动归档/合并 |
| 安装安全 | 无 | 静态代码扫描 + 按来源分级 |

### 学习能力

| 维度 | OpenClaw | Hermes |
|---|---|---|
| 自学习 | ❌ 无 | ✅ Background Review + Curator |
| 技能进化 | 手动更新 | 自动更新时间/内容维度的流转 |
| 记忆进化 | 无 | 从每次交互中提炼偏好 |

### 多 Agent

| 维度 | OpenClaw | Hermes |
|---|---|---|
| 多 Agent 支持 | ✅ 强项 | ✅ 支持但官方不推荐 |
| 通讯方式 | spawn / send（消息式）| delegate_task / kanban（看板式）|
| 团队协作 | ✅ Agent Team (开发中) | ❌ 不适合 |
| 上下文隔离 | 子 Agent 不继承主 session | 子 Agent 完全独立 |

### 生态

| 维度 | OpenClaw | Hermes |
|---|---|---|
| 插件数量 | 数千 | 内置为主 |
| 核心大小 | 极简，边界清晰 | 功能丰富但内聚 |
| 工具数量 | 通过插件扩展 | 60+ 内置 |
| Skill 市场 | ClawHub | 无（本地管理）|

### 前端

| 维度 | OpenClaw | Hermes |
|---|---|---|
| CLI | ✅ | ✅ |
| TUI | ✅ | ✅ |
| Web | ✅ | ✅ |
| 消息平台 | 微信/企微/飞书/Slack... | 20+ 平台 Gateway |
| 桌面应用 | ✅ | ❌ |
| 协议 | MCP, ACP | MCP, ACP |

## 2. 什么时候选 OpenClaw？

### 典型场景

```text
✅ 你需要多 Agent 协作
   例：planner 拆任务 → coder 实现 → reviewer 检查 → tester 验证

✅ 你需要多端精致体验
   例：电脑上用 CLI，手机上用企微，团队用 Web 控制台

✅ 你需要丰富的生态插件
   例：ClawHub 上有 TAPD/iWiki/工蜂/GitHub/Slack/Jira 插件

✅ 你的团队需要共用 Agent
   例：一个项目组共享一个技术方案 Reviewer Agent

✅ 你需要自定义工具和工作流
   例：对接企业内部系统，写自定义的 MCP Server
```

### 不适合的场景

```text
❌ 你只需要一个个人助手
   → Hermes 更轻量，OpenClaw 的生态是负担

❌ 你希望 Agent 自己学会你的工作习惯
   → OpenClaw 没有自学习能力

❌ 你的任务是长周期、跨多天、需要记忆的
   → OpenClaw 的记忆系统需要手动配置
```

## 3. 什么时候选 Hermes？

### 典型场景

```text
✅ 你需要一个"越用越懂你"的个人助手
   例：每天和同一个 Agent 工作，教它你的偏好

✅ 你的工作有重复模式
   例：每周的日志排查流程 → Agent 自动学会并沉淀为 Skill

✅ 你需要跨会话的记忆
   例：上次教的排查方法，下次自动调用

✅ 你主要个人使用，不做团队协作
   例：个人开发、个人研究、个人效率工具

✅ 你想研究"会学习的 Agent"本身
   例：研究自进化机制、记忆系统设计
```

### 不适合的场景

```text
❌ 你需要多 Agent 团队协作
   → Hermes 的多 Agent 能力有限，建议 OpenClaw

❌ 你的团队多人共用同一个 Agent
   → Hermes 的记忆是跨 profile 共享的，会互相污染

❌ 你需要大量第三方插件
   → Hermes 以内置工具为主，插件生态不如 OpenClaw
```

## 4. 混合使用：最佳实践

很多团队的实际做法是**两者互补**：

```text
┌──────── Hermes（个人助手）────────┐
│                                   │
│  日常问答、日志排查、个人效率       │
│  "越用越懂我"                    │
│                                   │
└───────────────────────────────────┘
              +
┌─────── OpenClaw（团队工作台）──────┐
│                                   │
│  多 Agent 协作、项目级任务          │
│  "一个人 = 一个团队"              │
│                                   │
└───────────────────────────────────┘
```

## 5. 快速选型决策树

```
你需要多 Agent 团队协作吗？
  ├─ 是 → OpenClaw
  └─ 否 → 你需要 Agent 自己学习你的工作习惯吗？
           ├─ 是 → Hermes
           └─ 否 → 你需要丰富的插件生态吗？
                    ├─ 是 → OpenClaw
                    └─ 否 → 两个都可以，试试看哪个体验更好
```

## 6. 一句话总结

> OpenClaw 是"我给你最多的工具，你要什么我都能接上"；Hermes 是"我记住你教我的每件事，越用越懂你"。选型取决于你是要"广"还是"深"。
