# 00 学习路线（打造 LLM 篇）

## 总体目标

把“理解大模型”升级为“能动手造大模型”。理解数据、架构、训练、推理四大支柱，并在小规模上完整实践一遍。

## 四大工程支柱

| 支柱 | 关键问题 | 对应章节 |
|---|---|---|
| 数据 | 喂什么、怎么清洗、怎么切 token | 04 |
| 架构 | 现代 LLM 相比原版 Transformer 改了什么 | 02, 03 |
| 训练 | 模型大到单卡放不下，怎么多卡训 | 05 |
| 推理 | 训练好后怎么又快又省地服务 | 06 |
| 工程基础 | 预算、流程、踩坑 | 01 |

## 建议节奏（约 8 周）

```text
第1周  AI 工程基础 + 用 nanoGPT 在小数据上训一个迷你 GPT
第2-3周 现代架构演进（RMSNorm/RoPE/GQA/SwiGLU/MoE）
第4周  数据工程：收集、清洗、去重、Tokenization(BPE)
第5-6周 多 GPU 训练：DP/TP/PP/ZeRO，混合精度
第7-8周 推理优化：KV Cache、量化、连续批处理、投机解码
```

## 学习原则

- 先小后大：用 nanoGPT 在莎士比亚文本上跑通，再谈规模化。
- 工程优先：这部分大量是“怎么省显存、怎么提吞吐”的工程问题，需结合 Part1 的 GPU/CUDA 知识。
- 数据决定上限：别只盯着模型结构，数据质量往往更关键。
- 每个优化都问：它省的是显存、算力还是通信？代价是什么？
