# 09 视频生成原理与 Wan 研究（含视频 Agent Demo）

> 对应你列出的：视频生成原理（上）、Wan 研究（完结篇）、视频制作 Agent Demo。

## 1. 视频生成难在哪

图像生成已经成熟，视频却难得多，因为多了**时间维度**：

```text
图像：[通道, 高, 宽]            一张静态图
视频：[帧数, 通道, 高, 宽]      一串图，且帧与帧之间必须连贯
```

核心挑战：

```text
1. 时间一致性：人物/物体在帧间不能忽大忽小、闪烁、变形
2. 运动合理性：要符合物理（球该往下落，不能乱飘）
3. 计算量爆炸：几十上百帧 × 高分辨率，显存和算力压力极大
4. 长程依赖：开头出现的物体，几秒后还要保持一致
```

## 2. 一句话直觉

> 视频生成 = 图像扩散 + “让相邻帧手拉手保持连贯”的时间建模。本质还是“从噪声去噪出内容”，只不过现在要同时对一段时间的所有帧一起去噪，并让它们在时间上彼此协调。

## 3. 模型怎么理解视频：时空建模

### 3.1 把视频切成时空块（patches/tokens）

现代视频模型（Sora、Wan）的关键：把视频压成**时空隐表示**，再切成一串 token，交给 Transformer 处理。

```text
原始视频 ──3D VAE/时空编码器──> 时空隐表示 ──切块──> 一串时空 token
                                                          │
                                              Diffusion Transformer (DiT)
                                                          │
                                               去噪 ──> 还原视频隐表示 ──解码──> 视频
```

这就是“视频版的潜空间扩散”：先压缩（含时间），在隐空间扩散，再解码。

### 3.2 时空注意力（Spatial-Temporal Attention）

Transformer 的注意力被拆成两类，让模型既懂“一帧内的空间关系”，又懂“跨帧的时间关系”：

```text
空间注意力：同一帧内不同区域互相关注（这一帧画得对不对）
时间注意力：不同帧的同一位置互相关注（帧之间是否连贯）
```

这是“模型理解视频”的核心机制——它同时在空间和时间两个轴上建模关系。

## 4. 主流技术路线

```text
图像模型 + 时间层：在预训练图像扩散模型里插入时间注意力层（如早期 AnimateDiff）
原生时空 DiT：    从头用 Diffusion Transformer 处理时空 token（Sora、Wan 路线，效果最好）
级联生成：        先生成低分辨率/低帧率，再超分 + 插帧
```

## 5. Wan 的研究要点（开源视频生成代表）

Wan（万相，阿里开源的视频生成模型系列）是当前**最值得学习的开源视频生成项目**之一，其设计体现了主流思路：

```text
1. 高压缩 3D 因果 VAE：把视频在时空上大幅压缩，降低扩散计算量，
   "因果"保证生成可流式、时间方向合理
2. Diffusion Transformer (DiT) 骨干：用 Transformer 在时空 token 上去噪
3. Flow Matching 训练目标：用前面学的流匹配，路径更直、训练高效
4. 文本/图像条件：支持文生视频(T2V)和图生视频(I2V)
5. 多分辨率/多时长训练：兼顾不同输出规格
```

学习价值：Wan 把本部分的知识点（VAE 压缩 + DiT + Flow Matching + 条件注入）全部串在了一起，是“内容生成篇”的集大成实践。

## 6. 视频制作 Agent Demo（把生成模型工程化）

单个视频模型只是“一次生成一小段”。要做一个**视频制作 Agent**，需要把生成模型放进一个 harness（呼应 `Harness_Engineering/`）：

```text
用户输入："做一个 30 秒的产品介绍视频，科技风，配字幕"
        │
   [规划 Agent] 拆成分镜脚本（scene 1..N，每段 prompt + 时长）
        │
   for each scene:
       [文生图/文生视频模型] 生成片段
       [一致性检查] 主体/风格是否跨片段一致（必要时重生成）
        │
   [拼接] 转场 + 配乐 + 字幕（调用 ffmpeg 等工具）
        │
   [评估] 时长/分辨率/内容是否符合要求（不达标则回到对应 scene）
        │
   输出成片
```

要点（和 Harness 篇一致）：

```text
- 工具化：文生视频、超分、插帧、配音、字幕、剪辑都封装成工具
- 状态管理：分镜脚本 = progress file，逐镜生成、可断点续跑
- 一致性 = 验证层：用参考帧/角色嵌入约束跨片段一致
- 人在环：关键分镜让人确认再继续
```

这就是“生成模型能力”如何通过 Agent harness 变成“可用的视频制作产品”。

## 7. 评估视频生成

```text
FVD (Fréchet Video Distance)：衡量生成视频与真实视频分布的差距
时间一致性指标：帧间一致性、光流平滑度
人工评估：清晰度、运动合理性、文字贴合度（仍是金标准）
```

## 本章小结

视频生成在图像扩散之上增加了时间维度，核心是“时空建模”——用 3D VAE 压缩时空、用 DiT 在时空 token 上去噪、用时空注意力保证帧间连贯，并越来越多采用 Flow Matching 训练。Wan 是把这些串起来的优秀开源代表。要做视频制作产品，还需把模型放进 Agent harness（分镜规划、工具调用、一致性验证、人在环）。这也标志着内容生成篇与 Agent 工程的交汇。
