# 01 机器学习基础

## 1. 什么是机器学习

传统编程通常是：

```text
规则 + 数据 -> 结果
```

机器学习更像是：

```text
数据 + 结果 -> 学到规则
```

例如，你想判断一封邮件是不是垃圾邮件。

传统方式是手写规则：

- 包含“免费中奖”就判为垃圾邮件。
- 包含很多链接就判为垃圾邮件。

但现实中规则太多、变化太快。机器学习的做法是给模型很多历史邮件和标签，让模型自己学习哪些特征和“垃圾邮件”有关。

## 2. 机器学习的核心组成

一个机器学习问题通常包含：

| 组成 | 含义 | 例子 |
|---|---|---|
| 数据 | 用来学习的样本 | 房屋面积、位置、价格 |
| 特征 | 描述样本的信息 | 面积、楼层、城市 |
| 标签 | 希望模型预测的答案 | 房价、是否违约 |
| 模型 | 从特征到标签的函数 | 线性回归、决策树、神经网络 |
| 损失函数 | 衡量预测错得多严重 | MSE、交叉熵 |
| 优化算法 | 调整模型参数的方法 | 梯度下降、Adam |
| 评估指标 | 衡量模型效果 | Accuracy、RMSE、AUC |

可以把机器学习看成一句话：

> 在数据中找到一个函数，让它在没见过的新数据上也能预测得好。

## 3. 监督学习、无监督学习、强化学习

### 3.1 监督学习

有标准答案的数据学习。

例子：

- 输入房屋信息，预测房价。
- 输入图片，判断是不是猫。
- 输入用户行为，预测是否会购买。

常见任务：

- 回归：预测连续数值，例如房价、温度、销量。
- 分类：预测类别，例如垃圾邮件/正常邮件、猫/狗。

### 3.2 无监督学习

没有标准答案，让模型自己发现结构。

例子：

- 用户分群。
- 文档聚类。
- 降维可视化。

常见方法：

- K-Means
- PCA
- t-SNE / UMAP

### 3.3 强化学习

智能体通过和环境交互，学习如何获得更高奖励。

例子：

- 游戏 AI
- 机器人控制
- 大模型对齐中的 RLHF

本部分只做概念铺垫，强化学习会放到后续部分系统学习。

## 4. 训练集、验证集、测试集

机器学习最容易犯的错误是：只在训练数据上表现好。

因此通常要把数据分成：

| 数据集 | 作用 |
|---|---|
| 训练集 | 用来训练模型参数 |
| 验证集 | 用来选择模型、调超参数 |
| 测试集 | 最后一次评估泛化能力 |

重要原则：

- 测试集不能参与训练。
- 测试集不能反复用来调参。
- 如果测试集被用来指导模型选择，它就不再是纯测试集。

## 5. 过拟合与欠拟合

### 欠拟合

模型太简单，连训练集都学不好。

例子：用一条直线拟合明显弯曲的数据。

表现：

- 训练误差高
- 验证误差高

解决方法：

- 换更复杂的模型
- 增加特征
- 减少过强正则化
- 训练更久

### 过拟合

模型太复杂，把训练数据中的噪声也记住了。

表现：

- 训练误差低
- 验证误差高

解决方法：

- 增加数据
- 降低模型复杂度
- 正则化
- 早停
- 数据增强
- 交叉验证

## 6. 泛化能力

泛化能力是机器学习最核心的目标：

> 模型不只是记住旧题，而是能做对新题。

一个模型如果训练集准确率 99%，测试集准确率 60%，说明它可能只是背题。

大模型也是如此。预训练损失下降不等于实际能力一定好，还需要各种验证集、基准测试和真实任务评估。

## 7. 损失函数与评估指标

损失函数用于训练，评估指标用于判断效果。二者不一定相同。

### 回归任务

常见损失与指标：

- MSE：均方误差，对大误差更敏感。
- MAE：平均绝对误差，更鲁棒。
- RMSE：MSE 开根号，单位与原标签一致。
- R²：解释方差比例。

### 分类任务

常见指标：

- Accuracy：准确率。
- Precision：预测为正的样本中有多少是真的正。
- Recall：真实正样本中有多少被找出来。
- F1：Precision 和 Recall 的平衡。
- ROC-AUC：排序能力。

如果正负样本极不均衡，只看 Accuracy 会很危险。

例如疾病筛查中，真实患病率只有 1%。一个模型永远预测“没病”，准确率也有 99%，但完全没用。

### benchmark（基准测试）

`benchmark` 可以理解为“统一考试”：用一套公开、固定、标准化的任务来比较不同模型的能力。

就像不同学生做同一张试卷，谁分高说明谁在这张卷子对应的能力上更强。机器学习里也是让不同模型在同一个数据集/任务上测试，比较 Accuracy、F1、AUC、Perplexity、BLEU 等指标。

常见 benchmark：

| Benchmark | 用途 |
|---|---|
| MNIST / CIFAR-10 / ImageNet | 图像分类 |
| GLUE / SuperGLUE | NLP 理解任务 |
| MMLU | 大模型多学科知识 |
| HumanEval | 代码生成 |
| GSM8K | 数学推理 |

它解决的问题是：不能只说“我的模型很好”，要有统一标准来比较。

但 benchmark 不是绝对真理，常见局限：

- 只代表某一类能力，单项高分不代表真实体验好。
- 可能被污染：如果测试数据出现在训练集里，模型相当于“背过题”，这就是后面会提到的 benchmark contamination。
- 容易刷榜：专门针对某个 benchmark 优化，泛化能力不一定强。
- 指标不能完全代表用户体验（是否有帮助、可靠、安全）。

所以更准确地说：benchmark 是评估模型能力的重要参考，但不能作为唯一标准。

## 8. 偏差-方差权衡

误差可以粗略理解为三部分：

```text
总误差 = 偏差 + 方差 + 不可避免噪声
```

### 偏差

模型假设太简单导致的系统性错误。

例如真实关系是曲线，却用直线拟合。

### 方差

模型对训练数据变化太敏感。

例如换一批训练数据，模型结果差很多。

### 权衡

- 简单模型：偏差高，方差低。
- 复杂模型：偏差低，方差高。

随机森林主要降低方差；Boosting 类方法通常降低偏差，但也可能过拟合。

## 9. 数据处理与特征工程

传统机器学习中，数据处理通常比模型选择更重要。

常见步骤：

- 缺失值处理
- 异常值检查
- 类别变量编码
- 数值标准化
- 日期时间特征
- 文本特征
- 特征交叉
- 数据泄漏检查

### 数据泄漏

数据泄漏是指训练时使用了真实预测时不可能获得的信息。

例子：预测用户是否会流失，却把“用户注销时间”作为特征。这会让离线指标很好，但上线后失效。

## 10. 机器学习项目标准流程

```text
定义问题
  -> 收集数据
  -> 探索性分析
  -> 划分数据集
  -> 数据预处理
  -> 选择基线模型
  -> 训练模型
  -> 评估模型
  -> 调参
  -> 错误分析
  -> 部署或总结
```

不要一开始就追求复杂模型。先做一个简单 baseline，确认数据和指标可靠。

## 11. 与大模型学习的关系

这些基础会在大模型中反复出现：

| 机器学习基础 | 大模型中的对应问题 |
|---|---|
| 损失函数 | next token prediction 的交叉熵 |
| 泛化能力 | 模型是否能处理未见任务 |
| 数据泄漏 | benchmark contamination |
| 过拟合 | 训练集记忆、评估失真 |
| 优化 | AdamW、学习率调度、warmup |
| 指标 | perplexity、BLEU、ROUGE、人类偏好 |
| 数据处理 | tokenization、去重、过滤、质量控制 |

## 本章小结

机器学习不是“调包跑模型”，而是围绕数据、模型、损失、优化和评估构建一个可靠系统。后续所有算法都可以放进这个框架中理解。
