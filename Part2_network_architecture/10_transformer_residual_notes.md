## 1. 为什么残差连接是深层网络的“生命线”

在 `08_resnet.md` 中你已经学过残差连接：

```text
y = x + F(x)
```

直觉是：如果 `F(x)` 学不好，网络至少还能走恒等路径 `x`，不会越变越差。

KM 中关于 Transformer 残差演进的文章进一步强调：

> 残差连接已经不只是“加一下”，而是在深层 Transformer 里承担“信息流控制系统”的角色。

深层网络遇到的核心问题：

```text
1. 梯度消失/爆炸：层数越深，反向传播越难
2. 信息稀释：早期层的信息在不断叠加中被冲淡
3. 尺度漂移：每层输出分布变化，训练不稳定
4. 深度扩展：想训练 100+ 甚至 1000 层模型时，普通残差不够用
```

## 2. Transformer 残差的三代演进

### 2.1 第一代：Post-LN（原始 Transformer）

原始 Transformer 结构：

```text
x -> Sublayer -> Add -> LayerNorm
```

公式：

```text
x_{l+1} = LN(x_l + F(x_l))
```

优点：输出分布稳定。

缺点：梯度路径上会经过 LayerNorm，深层训练时梯度不够顺畅。

### 2.2 第二代：Pre-LN（现代 LLM 主流）

现代大模型常用：

```text
x -> LayerNorm -> Sublayer -> Add
```

公式：

```text
x_{l+1} = x_l + F(LN(x_l))
```

优点：恒等路径更干净，梯度可以沿着 `x_l` 直接回传。

这也是 GPT、LLaMA 等大模型常用 Pre-LN 的原因。

### 2.3 第三代：DeepNorm / ReZero / Attention Residuals

当模型更深时，仅靠 Pre-LN 仍不够，于是出现了更复杂的残差控制方式。

#### DeepNorm

```text
x_{l+1} = LN(α x_l + F(x_l))
```

通过缩放主路径 `α` 和特殊初始化，让非常深的 Transformer 也能训练。

#### ReZero

```text
x_{l+1} = x_l + α F(x_l)
```

其中 `α` 初始化为 0。模型一开始几乎是恒等映射，再逐渐学习每层该改多少。

#### Attention Residuals

更激进的想法：不要总是简单相加，而是让模型对历史层做注意力选择。

```text
当前层不只接收上一层，而是从多个历史层中按权重选择信息。
```

这解决了 Pre-LN 的一个问题：早期层信息可能被后续层反复“冲淡”。

## 3. 残差连接的本质：不是“加法”，而是“保留旧信息 + 控制更新幅度”

可以把每层 Transformer 想象成一个编辑器：

```text
输入 x = 当前文档
F(x) = 这一层想做的修改
x + F(x) = 保留原文，同时加入修改
```

如果没有残差，模型每层都像“重写全文”；有残差后，每层只需要学“增量修改”。

这非常重要，因为深层网络越深，越应该让每层承担小而稳定的修改，而不是让每层都重新构造表示。

---

## 4. Transformer 从专用网络中胜出：归纳偏置的变化

CNN 和 RNN 都有强归纳偏置：

| 网络 | 归纳偏置 | 擅长 |
|---|---|---|
| CNN | 局部性 + 平移不变性 | 图像局部纹理 |
| RNN | 时间顺序递推 | 序列历史依赖 |
| Transformer | 任意位置两两交互 | 文本、多模态、图像、视频 |

早期 CNN/RNN 的优势来自“架构替人类写死了一些假设”。但数据和算力变大后，Transformer 通过注意力机制学习关系，变得更通用。

这也是 DiT（Diffusion Transformer）逐渐替代 U-Net 的底层原因：

```text
当图像生成不再只是局部纹理合成，而是需要全局语义、长距离关系、多模态条件时，Transformer 的全局建模优势开始超过 CNN/U-Net 的局部偏置。
```

## 5. 真实业务案例：从 XGBoost 到多模态 Transformer

KM 中微信支付商户可信度评分案例展示了一个典型演进路线：

```text
第一代：XGBoost
  - 结构化特征强
  - 可解释性好
  - 但对自由文本、多模态输入能力弱

第二代：LLM SFT
  - 把可信度评分重新定义为“有序分类代理排序任务”
  - 能处理结构化字段 + 自由文本
  - 但推理成本高

第三代：多模态 FT-Transformer
  - 文本 Embedding + 数值特征 + 类别特征做跨模态自注意力
  - 兼顾表达能力和推理吞吐
```

指标提升：

```text
KS:  0.611 → 0.711 → 0.779
AUC: 0.908 → 0.936 → 0.952
```

这说明 Transformer 不只是“语言模型的积木”，也可以作为结构化 + 文本 + 多模态特征融合的通用骨架。

## 6. 本阶段补充后的学习重点

学完 Part2 后，你应该能回答：

```text
1. CNN 为什么擅长图像？因为局部性和平移不变性。
2. RNN 为什么被 Transformer 替代？因为长程依赖和并行效率弱。
3. Transformer 为什么强？因为任意位置之间可以直接交互。
4. 残差为什么关键？因为它提供梯度高速路和信息保留路径。
5. Pre-LN 为什么成为主流？因为梯度路径更干净。
6. 工业模型为什么会从 XGB 进化到 Transformer？因为输入从结构化字段扩展到了文本、多模态和复杂关系。
```
