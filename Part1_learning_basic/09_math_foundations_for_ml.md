# 09 机器学习数学基础速查

本章不是为了把数学学成考试科目，而是为了让你在看机器学习、深度学习和大模型时，不会被符号挡住。目标是：看到公式能知道它在表达什么，写代码时知道张量形状为什么这样设计。

## 1. 学数学的正确目标

学习大模型前，数学不需要一开始追求严密证明，更重要的是掌握四件事：

| 数学主题 | 在机器学习里的作用 |
|---|---|
| 线性代数 | 描述向量、矩阵、张量和神经网络层 |
| 概率统计 | 描述不确定性、数据分布、评估和采样 |
| 微积分 | 描述损失如何随参数变化，理解梯度 |
| 优化 | 用可计算的方法让损失变小 |

一句话：

> 线性代数负责“怎么表示”，概率统计负责“怎么理解数据”，微积分负责“怎么计算变化”，优化负责“怎么让模型变好”。

## 2. 线性代数：从表格到张量

### 2.1 标量、向量、矩阵、张量

| 对象 | 直觉 | 例子 |
|---|---|---|
| 标量 | 一个数 | 一个 loss 值 |
| 向量 | 一串数 | 一个样本的特征、一个词向量 |
| 矩阵 | 一张二维表 | 一批样本的特征矩阵 |
| 张量 | 更高维数组 | 图片 batch、文本 embedding |

机器学习常把数据写成：

```text
X: [样本数, 特征数]
w: [特征数]
y_hat = Xw
```

这就是线性回归、逻辑回归和神经网络全连接层的基础。

### 2.2 矩阵乘法的直觉

矩阵乘法不是随便把数相乘，而是在做“加权求和”。

例如：

```text
预测房价 = 面积权重 × 面积 + 楼层权重 × 楼层 + 城市权重 × 城市特征 + 偏置
```

神经网络中的 `Linear` 层也是类似思想：

```text
输出 = 输入 × 权重矩阵 + 偏置
```

区别只是神经网络会把这个过程堆很多层，并加入非线性激活函数。

### 2.3 shape 是理解深度学习的关键

看 PyTorch 代码时优先问：

```text
这个张量的 shape 是什么？
这一维表示 batch、特征、通道，还是序列长度？
矩阵乘法前后维度是否对得上？
```

常见 shape：

```text
表格数据: [batch_size, num_features]
图片: [batch_size, channels, height, width]
文本 token: [batch_size, seq_len]
文本 embedding: [batch_size, seq_len, hidden_size]
Transformer 注意力分数: [batch_size, num_heads, seq_len, seq_len]
```

后续学大模型时，很多问题其实都是 shape 问题。

## 3. 概率统计：模型不是绝对确定的规则

### 3.1 概率的直觉

概率描述不确定性。模型输出的很多结果都可以理解为概率或分数。

例如二分类模型输出：

```text
p = 0.8
```

它不是说“绝对是正类”，而是说“模型认为属于正类的概率或置信度较高”。

### 3.2 分布

数据不是孤立的点，而是来自某种分布。

机器学习真正希望做到的是：

```text
在训练数据分布上学到规律，并能泛化到未来相似分布的数据。
```

如果训练数据和真实使用场景差异很大，就会出现分布偏移。例如用城市白天道路数据训练自动驾驶模型，却在夜晚雨天道路上使用，效果可能明显下降。

### 3.3 均值、方差与标准差

| 概念 | 含义 |
|---|---|
| 均值 | 数据的平均水平 |
| 方差 | 数据围绕均值波动的程度 |
| 标准差 | 方差开根号，和原始数据同单位 |

标准化常用：

```text
x_standardized = (x - mean) / std
```

这会让不同特征处在相近尺度，方便线性模型和神经网络训练。

### 3.4 期望

期望可以理解为“长期平均结果”。

损失函数常写成期望形式：

```text
希望模型在所有可能样本上的平均损失最小
```

现实中无法拿到所有样本，所以用训练集或 mini-batch 估计这个平均损失。

### 3.5 最大似然与交叉熵

最大似然的直觉是：

> 调整模型参数，让已经发生的训练数据看起来最合理。

分类任务常用交叉熵，本质上是在惩罚模型给真实类别太低概率。

大语言模型预测下一个 token 也是类似：如果正确 token 是“学习”，模型却给它很低概率，loss 就会很大。

## 4. 微积分：梯度告诉模型该往哪里改

### 4.1 导数

导数描述一个量变化时，另一个量变化得有多快。

例如：

```text
loss 随 w 变化得快不快？
```

如果改变某个参数会让 loss 明显变化，这个参数的梯度就大。

### 4.2 偏导数

模型通常有很多参数，偏导数表示“只看某一个参数变化时，loss 如何变化”。

```text
∂loss / ∂w1
```

可以理解为：如果只动 `w1`，损失会怎么变。

### 4.3 梯度

梯度是所有偏导数组成的向量。

```text
gradient = [∂loss/∂w1, ∂loss/∂w2, ..., ∂loss/∂wn]
```

它指向 loss 上升最快的方向。训练时要让 loss 下降，所以往负梯度方向更新参数：

```text
参数 = 参数 - 学习率 × 梯度
```

### 4.4 链式法则与反向传播

神经网络是很多函数套在一起：

```text
输入 -> 第 1 层 -> 第 2 层 -> 损失
```

链式法则告诉我们如何计算“前面参数”对最终 loss 的影响。

反向传播就是链式法则在神经网络中的高效实现。PyTorch 的 `autograd` 会自动完成这件事。

## 5. 优化：训练模型就是最小化损失

### 5.1 损失函数地形

可以把损失函数想象成一片山地：

- 高处表示 loss 大。
- 低处表示 loss 小。
- 训练就是从当前位置一步步往低处走。

### 5.2 梯度下降三种常见形式

| 方法 | 每次用多少数据算梯度 | 特点 |
|---|---|---|
| Batch Gradient Descent | 全部训练集 | 稳定但慢 |
| Stochastic Gradient Descent | 单个样本 | 噪声大但更新快 |
| Mini-batch SGD | 一小批样本 | 深度学习最常用 |

PyTorch 的 `DataLoader(batch_size=...)` 就是在组织 mini-batch。

### 5.3 学习率为什么重要

学习率太小：走得慢。

学习率太大：可能越过谷底，甚至发散。

```text
合适学习率：loss 稳定下降
过大学习率：loss 震荡或 NaN
过小学习率：loss 几乎不动
```

### 5.4 局部最优、鞍点与深度学习

传统优化里常担心局部最优。深度学习中参数很多，真实情况更复杂，常见问题还包括：

- 鞍点：某些方向像谷底，某些方向像山顶。
- 梯度消失：梯度太小，学不动。
- 梯度爆炸：梯度太大，训练不稳定。
- 损失曲面复杂：不同初始化可能得到不同结果。

所以现代训练会使用 AdamW、学习率调度、归一化、残差连接、梯度裁剪等技巧。

## 6. 数值稳定性：代码里经常踩的坑

数学公式写起来很漂亮，但计算机有精度限制。

常见问题：

| 问题 | 例子 | 解决思路 |
|---|---|---|
| 溢出 | `exp(1000)` 太大 | 使用稳定实现，如 log-sum-exp |
| 下溢 | 极小概率变成 0 | 用 log probability |
| 除零 | 标准差为 0 | 加很小的 `epsilon` |
| dtype 不合适 | float16 训练不稳定 | mixed precision、梯度缩放 |

学习 PyTorch 时要经常检查：

```text
shape、dtype、device、是否出现 NaN/Inf
```

## 7. 你需要掌握到什么程度

Part1 阶段不要求证明定理，但建议达到以下标准：

- 能解释向量、矩阵、张量的区别。
- 能看懂 `Xw + b` 的含义。
- 能说明为什么标准化有用。
- 能解释概率、分布、均值、方差、期望。
- 能说明交叉熵为什么惩罚“给正确答案低概率”。
- 能用通俗语言解释梯度下降。
- 能说清楚 `loss.backward()` 大概做了什么。
- 能排查常见的 `shape`、`dtype`、`device` 错误。

## 8. 推荐资料

- Gilbert Strang, *Introduction to Linear Algebra*：线性代数经典教材。
- Deisenroth, Faisal, Ong, *Mathematics for Machine Learning*：专门面向机器学习的数学教材。
- Bishop, *Pattern Recognition and Machine Learning*：概率视角更系统。
- Goodfellow, Bengio, Courville, *Deep Learning*：第 2、3、4 章适合作为数学复习。
- Khan Academy / 3Blue1Brown：适合建立线性代数和微积分直觉。
- Stanford CS229 Linear Algebra and Probability Review Notes。

## 本章小结

数学是工具，不是门槛。先抓住直觉：数据用向量和矩阵表示，模型输出带有不确定性，损失函数衡量错误，梯度告诉参数如何更新，优化算法负责一步步降低损失。掌握这些，再学习深度学习和大模型会顺畅很多。
