# 04 Ensemble：随机森林与 GBDT

集成学习的核心思想是：

> 一个模型可能不稳定，但很多模型组合起来，往往更稳、更强。

就像一个人做判断可能有偏见，但多个不同的人投票，结果通常更可靠。

## 1. 什么是集成学习

集成学习把多个弱模型组合成一个强模型。

常见方式：

| 方法 | 思想 | 代表模型 |
|---|---|---|
| Bagging | 多个模型并行训练，减少方差 | 随机森林 |
| Boosting | 模型按顺序训练，逐步修正错误 | AdaBoost、GBDT、XGBoost |
| Stacking | 多个模型的输出再交给另一个模型学习 | 模型融合 |

本章重点是随机森林和 GBDT。

## 2. Bagging

Bagging 是 Bootstrap Aggregating 的缩写。

步骤：

1. 从原始训练集中有放回抽样，构造多个训练子集。
2. 每个子集训练一个模型。
3. 分类任务投票，回归任务取平均。

有放回抽样意味着：

- 一个样本可能被抽中多次。
- 一些样本可能没有被抽中。

这种做法能让每个模型看到的数据略有不同，从而降低整体方差。

## 3. 随机森林

随机森林 = Bagging + 决策树 + 特征随机选择。

它的随机性来自两处：

1. 样本随机：每棵树使用 bootstrap 样本。
2. 特征随机：每次分裂只从部分特征中选择最优分裂。

这会让不同树之间差异更大，投票或平均后更稳定。

## 4. 随机森林为什么有效

单棵决策树容易过拟合，方差高。

随机森林通过很多棵树平均，降低方差：

```text
单棵树：可能因为训练数据变化而大幅变化
森林：多个不同树的结果平均，波动变小
```

但前提是树之间不能太相似。如果所有树都一样，平均也没有意义。因此随机森林要引入样本随机和特征随机。

## 5. 随机森林的重要参数

| 参数 | 含义 |
|---|---|
| `n_estimators` | 树的数量 |
| `max_depth` | 每棵树最大深度 |
| `max_features` | 每次分裂可考虑的特征数 |
| `min_samples_leaf` | 叶子节点最少样本数 |
| `bootstrap` | 是否使用有放回抽样 |

一般来说：

- 树越多，效果更稳定，但训练更慢。
- 树越深，拟合能力越强，但更容易过拟合。
- `max_features` 越小，树之间差异越大。

## 6. Boosting

Boosting 的思想是：

> 不一次性训练一个完美模型，而是一步步训练很多弱模型，每一步都修正前面的不足。

与 Bagging 不同：

- Bagging 中的模型可以并行训练。
- Boosting 中的模型通常按顺序训练。

## 7. GBDT 的直觉

GBDT 是 Gradient Boosting Decision Tree。

它使用一系列决策树逐步改进预测。

可以这样理解：

```text
第 1 棵树：先给一个粗略预测
第 2 棵树：学习第 1 棵树的错误
第 3 棵树：继续修正剩余错误
...
最终预测 = 所有树的结果相加
```

回归任务中，最简单理解是后一棵树拟合残差。

例如真实值是 100：

```text
初始预测：60，残差 40
第 2 步补 30，剩余残差 10
第 3 步补 8，剩余残差 2
```

模型逐步逼近真实值。

## 8. 为什么叫 Gradient Boosting

GBDT 不只是拟合普通残差，而是在损失函数的负梯度方向上改进模型。

直觉：

- 损失函数告诉模型错得多严重。
- 梯度告诉模型应该朝哪个方向改。
- 新树拟合这个改进方向。

所以 GBDT 可以支持不同损失函数，不只适用于 MSE。

## 9. GBDT 预测形式

GBDT 是加法模型：

```text
F(x) = f0(x) + η*f1(x) + η*f2(x) + ... + η*fm(x)
```

其中：

- `f0` 是初始模型。
- `fi` 是第 i 棵树。
- `η` 是学习率。

学习率越小，每棵树贡献越小，通常需要更多树，但泛化能力可能更好。

## 10. GBDT 的重要参数

| 参数 | 含义 |
|---|---|
| `n_estimators` | 树的数量 |
| `learning_rate` | 每棵树贡献大小 |
| `max_depth` | 单棵树深度 |
| `subsample` | 样本采样比例 |
| `min_samples_leaf` | 叶子节点最少样本数 |

常见经验：

- `learning_rate` 小一些，`n_estimators` 大一些，通常更稳。
- 树不要太深，GBDT 常使用浅树。
- 需要验证集或交叉验证控制过拟合。

## 11. XGBoost、LightGBM、CatBoost

它们都是 GBDT 思想的高性能工程实现或改进。

| 模型 | 特点 |
|---|---|
| XGBoost | 正则化强、工程优化好、经典竞赛利器 |
| LightGBM | 训练快、内存省，适合大规模数据 |
| CatBoost | 对类别特征处理友好，减少目标泄漏 |

### XGBoost

经典论文是 Chen and Guestrin 的 *XGBoost: A Scalable Tree Boosting System*，KDD 2016。

它在目标函数中显式加入正则项，并做了很多系统优化。

### LightGBM

Microsoft 提出，常用直方图算法、Leaf-wise 生长策略、GOSS 和 EFB 等技术提升效率。

### CatBoost

Yandex 提出，对类别特征和 ordered boosting 做了重要改进。

## 12. 随机森林 vs GBDT

| 维度 | 随机森林 | GBDT |
|---|---|---|
| 训练方式 | 多棵树并行 | 多棵树串行 |
| 核心思想 | 降低方差 | 降低偏差，逐步修正错误 |
| 对调参敏感度 | 较低 | 较高 |
| 过拟合风险 | 相对低 | 需要控制 |
| 速度 | 容易并行 | 串行依赖更强 |
| 表格数据效果 | 稳定 | 通常更强 |

## 13. 线性回归、决策树、GBDT 的关系

你原始列表中把“线性回归，决策树，GBDT”放在一起，这是非常合理的对比组合。

| 模型 | 学到的函数形态 | 适合做什么 |
|---|---|---|
| 线性回归 | 线性关系 | baseline、解释性强 |
| 决策树 | 分段规则 | 理解非线性与规则切分 |
| GBDT | 多棵树的加法模型 | 表格数据强模型 |

建议学习时用同一个数据集比较这三个模型，而不是分别孤立学习。

## 14. 什么时候用 GBDT，什么时候用深度学习

经验上：

- 表格数据：GBDT 往往非常强。
- 图像、语音、文本、序列、多模态：深度学习更强。
- 数据量不大且特征结构清晰：传统机器学习更容易训练和解释。
- 数据极大且模式复杂：深度学习优势明显。

大模型主要处理文本、图像、语音、多模态等复杂数据，因此深度学习是主线。但在很多业务预测、风控、推荐特征处理场景，GBDT 仍然非常重要。

## 本章小结

随机森林通过“多个不同树投票”降低单棵树的不稳定性。GBDT 通过“一棵接一棵修正错误”构建强模型。理解这两类方法，有助于建立偏差、方差、加法模型和迭代优化的直觉。
