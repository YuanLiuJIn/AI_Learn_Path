# 02 强化学习数学基础

## 1. 本章目标

把上一章的直觉变成可计算的数学框架：MDP、价值函数、贝尔曼方程。这些是后面所有算法的地基。别怕公式——每个都有简单直觉。

## 2. MDP：马尔可夫决策过程

RL 问题通常建模成 MDP，由五元组定义：

```text
MDP = (S, A, P, R, γ)
S：状态集合
A：动作集合
P：状态转移概率 P(s'|s,a)（在 s 做 a，转到 s' 的概率）
R：奖励函数 R(s,a)
γ：折扣因子
```

**马尔可夫性质**（关键假设）：

> 未来只取决于当前状态，与如何到达当前状态无关。
> “当前状态已包含决策所需的全部信息。”

```text
下棋：当前棋盘局面就够了，不需要知道之前怎么走到这个局面的。
```

## 3. 策略 π

策略是“状态 → 动作”的规则：

```text
确定性策略：π(s) = a            （状态 s 一定选动作 a）
随机策略：  π(a|s) = 概率        （状态 s 下各动作的概率分布）
```

RL 的目标就是找到**最优策略 π\***，使长期回报最大。

## 4. 价值函数：衡量“好不好”

### 4.1 状态价值 V(s)

```text
V^π(s) = 从状态 s 出发，遵循策略 π，未来能拿到的期望回报
       = E[ G_t | s_t = s ]
```

直觉：“处在这个状态，按这套策略走下去，大概能得多少分。”

### 4.2 动作价值 Q(s, a)

```text
Q^π(s,a) = 在状态 s 先做动作 a，之后遵循 π，能拿到的期望回报
```

直觉：“在这个状态，先做这个动作好不好。” Q 比 V 多了“当前动作”这个维度，对“选动作”更直接有用。

### 4.3 V 和 Q 的关系

```text
V^π(s) = Σ_a π(a|s) · Q^π(s,a)     （V 是各动作 Q 的加权平均）
```

## 5. 贝尔曼方程：价值的递归结构

强化学习最核心的方程。直觉一句话：

> 当前状态的价值 = 即时奖励 + 折扣后的下一状态价值。

```text
V^π(s) = E[ r + γ·V^π(s') ]

Q^π(s,a) = E[ r + γ·Σ_{a'} π(a'|s')·Q^π(s',a') ]
```

它把“算无限长的未来回报”这个难题，变成“当前奖励 + 下一步的价值”的递归——这是几乎所有 RL 算法的计算基础。

## 6. 最优价值与最优策略

最优策略对应最优价值函数，满足**贝尔曼最优方程**：

```text
Q*(s,a) = E[ r + γ·max_{a'} Q*(s',a') ]
               └── 下一步选最好的动作 ──┘

最优策略：在每个状态选 Q* 最大的动作
  π*(s) = argmax_a Q*(s,a)
```

注意这里是 `max`（选最优动作），不再是按策略平均——这是“控制”（找最优策略）的关键。

## 7. 怎么求解：三条路线（预告后续章节）

```text
1. 已知环境模型 P、R -> 动态规划（价值迭代/策略迭代），但现实常不知道模型
2. 不知道模型，靠采样 -> 蒙特卡罗 / TD（第 3 章）
3. 状态太多无法用表格 -> 神经网络逼近（DeepRL，第 5 章起）
```

## 8. 一个最小网格世界例子

```text
3×3 网格，右下角是终点(奖励+1)，每走一步奖励 -0.1（鼓励快速到达）
状态 = 格子位置，动作 = 上下左右
价值迭代会算出：离终点越近的格子 V 越高
最优策略：每个格子都朝"V 更高的邻居"走
```

手算一遍这个小例子，贝尔曼方程就不再抽象。

## 本章小结

MDP 用 (S, A, P, R, γ) 形式化决策问题，核心假设是马尔可夫性。价值函数 V(s)/Q(s,a) 衡量“状态/动作有多好”，贝尔曼方程给出“当前价值 = 即时奖励 + 折扣的未来价值”的递归结构，贝尔曼最优方程定义最优策略。后续算法都是在“不知道模型/状态太多”等现实约束下求解它。
