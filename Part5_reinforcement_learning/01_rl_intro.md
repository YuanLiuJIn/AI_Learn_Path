# 01 强化学习基础入门

## 1. 它解决什么问题

监督学习需要“标准答案”（每个输入配一个标签）。但很多问题没有标准答案，只有“做得好不好”的反馈：

```text
下棋：没人告诉你"这一步的标准答案"，只有最后输赢
打游戏：没有标签，只有得分
机器人走路：没人标注每个关节角度，只有"走没走稳"
```

强化学习（Reinforcement Learning, RL）解决的就是这类**序列决策**问题：

> 智能体（agent）在环境中不断行动，根据获得的奖励，学会一套“在什么状态该做什么动作”的策略，以最大化长期回报。

## 2. 一句话直觉

> RL 就是“试错学习”：像训练小狗，做对了给奖励，做错了没奖励/惩罚，反复多次，它就学会了该怎么做。

## 3. 核心要素（先建立词汇表）

```text
Agent（智能体）：做决策的主体（棋手、游戏 AI、机器人）
Environment（环境）：智能体所处的世界（棋盘、游戏、物理世界）
State s（状态）：当前环境的描述（棋盘局面、游戏画面）
Action a（动作）：智能体能做的选择（落子、按键、转向）
Reward r（奖励）：环境给的反馈信号（得分、输赢）
Policy π（策略）：在状态 s 下选动作 a 的规则（智能体的"大脑"）
```

交互循环：

```text
       动作 a_t
Agent ─────────> Environment
   ^                  │
   │ 状态 s_{t+1}      │
   └── 奖励 r_{t+1} ───┘

智能体观察状态 -> 选动作 -> 环境给新状态和奖励 -> 循环
```

## 4. RL 与监督学习的根本区别

| 维度 | 监督学习 | 强化学习 |
|---|---|---|
| 反馈 | 每个样本有正确标签 | 只有奖励信号（可能延迟、稀疏）|
| 数据 | 固定数据集 | 智能体自己与环境交互产生 |
| 目标 | 拟合标签 | 最大化长期累积奖励 |
| 影响 | 样本独立 | 当前动作影响未来状态（序列依赖）|

## 5. 两个核心难点

### 5.1 延迟奖励与信用分配

```text
下棋第 10 步走错，但要到第 50 步才输。
到底是哪一步的错？—— 这就是"信用分配"难题。
RL 要学会把最终的输赢"归功/归咎"到中间的关键动作上。
```

### 5.2 探索与利用（Exploration vs Exploitation）

```text
利用(Exploitation)：选目前已知最好的动作（吃常去的那家好吃的餐厅）
探索(Exploration)：尝试没试过的动作（万一发现更好的新餐厅）
两者要平衡：只利用会错过更优解，只探索学不到东西。
常用 ε-greedy：大部分时间选最优，偶尔(ε 概率)随机探索。
```

## 6. 回报（Return）：为什么要“折扣”

智能体要最大化的不是单步奖励，而是**长期累积回报**：

```text
G_t = r_{t+1} + γ·r_{t+2} + γ²·r_{t+3} + ...

γ（折扣因子，0~1）：未来的奖励打折
  γ 接近 0：只看眼前（短视）
  γ 接近 1：重视长远
```

折扣的意义：未来不确定，且让数学收敛。

## 8. 代码直觉（与环境交互的骨架）

```python
import gymnasium as gym

env = gym.make("CartPole-v1")          # 经典平衡杆环境
state, _ = env.reset()
total = 0
for _ in range(500):
    action = env.action_space.sample() # 这里先随机选动作（后面章节学会智能选）
    state, reward, terminated, truncated, _ = env.step(action)
    total += reward
    if terminated or truncated:
        break
print("回合总奖励:", total)
```

后续章节就是要把 `env.action_space.sample()`（随机）换成“学出来的聪明策略”。

## 本章小结

强化学习是“试错学习”：智能体通过与环境交互获得奖励，学习最大化长期回报的策略。核心要素是状态、动作、奖励、策略；核心难点是延迟奖励的信用分配与探索-利用平衡。它与监督学习的根本区别在于“没有标签、只有奖励，且动作影响未来”。下一章建立其数学框架 MDP。
