# 03 基础 RL（上）：蒙特卡罗与 TD

## 1. 它解决什么问题

贝尔曼方程需要知道环境模型（转移概率 P、奖励 R）。但现实中我们**通常不知道模型**——不知道下棋每一步对手会怎么走、不知道游戏内部规则。

> 怎么在“不知道环境规则”的情况下，仅靠和环境交互的经验，估计出价值函数？

两种经典思路：蒙特卡罗（MC）和时序差分（TD）。

## 2. 蒙特卡罗方法（Monte Carlo, MC）

### 一句话直觉

> 玩完一整局，看最终拿了多少分，用这个真实回报去更新沿途每个状态的价值。“先玩完，再复盘。”

### 做法

```text
1. 用当前策略玩一整个回合（episode），直到结束
2. 算出每个状态的真实回报 G（从该状态到回合结束的累积奖励）
3. 用 G 更新该状态的价值估计（取多次回合的平均）

V(s) ← V(s) + α·(G - V(s))
       用"真实回报 G"修正"当前估计 V(s)"
```

### 特点

```text
优点：无偏（用的是真实回报），概念简单
缺点：必须等回合结束才能更新；回报方差大；不适合连续不终止的任务
```

## 3. 时序差分（Temporal Difference, TD）

### 一句话直觉

> 不用等回合结束，每走一步就更新：用“即时奖励 + 对下一状态的估计”来修正当前估计。“边玩边学，走一步学一步。”

### TD(0) 更新

```text
V(s) ← V(s) + α·[ r + γ·V(s') - V(s) ]
                  └──── TD 目标 ────┘
                  └──── TD 误差 δ ────────┘

r + γ·V(s')：用即时奖励 + 下一状态的现有估计，作为更新目标
δ = r + γ·V(s') - V(s)：TD 误差，衡量"惊喜程度"
```

这其实是把贝尔曼方程 `V(s)=E[r+γV(s')]` 变成了可采样的更新规则。

### 关键概念：自举（Bootstrapping）

```text
TD 用"自己对下一状态的估计 V(s')"来更新 V(s) —— 用估计更新估计，叫自举。
MC 不自举（用真实回报）。
```

## 4. MC vs TD 对比

| 维度 | 蒙特卡罗 (MC) | 时序差分 (TD) |
|---|---|---|
| 何时更新 | 回合结束后 | 每一步 |
| 用什么更新 | 真实回报 G | r + γV(s')（自举）|
| 偏差 | 无偏 | 有偏（依赖估计）|
| 方差 | 高（整局累积）| 低（只看一步）|
| 适用 | 必须有终止的回合 | 连续/不终止任务也行 |
| 学习速度 | 慢 | 通常更快 |

直觉记忆：

```text
MC：考完试看总分再复盘（准但慢，要等结束）
TD：每做一题就根据"这题对不对 + 估计后面能得多少"调整（快，但依赖估计）
```

## 5. n-step 与 TD(λ)：两者的统一

MC 和 TD 是一个谱的两端：

```text
TD(0)：只看 1 步
n-step TD：看 n 步再自举
MC：看到回合结束（n = ∞）
TD(λ)：用 λ 加权综合所有 n-step，平衡偏差与方差
```

## 6. 代码直觉（TD(0) 估计价值）

```python
def td0_update(V, s, r, s_next, alpha=0.1, gamma=0.99, done=False):
    target = r if done else r + gamma * V[s_next]   # TD 目标
    V[s] += alpha * (target - V[s])                 # 朝目标修正
    return V

# 在环境交互循环里，每走一步就调用一次（无需等回合结束）
```

## 7. 为什么 TD 如此重要

```text
- 在线学习：边交互边更新，效率高
- 适合连续任务（没有明确终点）
- 是后续 SARSA、Q-Learning、DQN 的核心机制（它们都基于 TD）
```

## 本章小结

在不知道环境模型时，MC 用“整局真实回报”更新价值（无偏但方差大、要等结束），TD 用“即时奖励 + 下一状态估计”每步更新（自举、低方差、更快）。n-step / TD(λ) 把两者统一成一个谱。TD 是后续所有控制算法（SARSA、Q-Learning、DQN）的基础。
